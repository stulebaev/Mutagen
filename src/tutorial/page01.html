General Local Parallel computational scheme

  • User launches DI GUI on one of the nodes - Master Host (HASP Key should be plugged in it)
  • Using DI GUI user starts MPI executable on Master Host.
  • MPI executable starts a set of DI's computational modules on all selected cluster nodes.
  • Using DI GUI user controls computational progress and save results after computation is completed.

Supported MPI packages

Currently DI supports 3 MPI packages:

  1. DeinoMPI (starting from version 1.1.0) for Windows.
  2. MPICH2 (starting from version 1.0.6) for Windows.
  3. LAM/MPI (starting from version 7.1.4) for Linux and MacOS.

MPICH1 for Linux and Windows is no longer supported in two reasons: 1. MPICH1 does not support necessary part of MPI-2 standard. 2. MPICH1 is not supported by its manufacturers.

Troubleshooting

In some cases computation can be aborted. It can occure just after start. The most probable reasons of failure are:

  1. Some settings are not specified correctly in DI's Configuration Dialog.
  2. There is some Antivirus software installed on one of cluster nodes that blocks network activity.
  3. There is some Firewall software installed on one of cluster nodes that blocks network activity.
  4. MPICH2 or LAM/MPI software is not installed correctly on local cluster.
  5. Network connection between 2 or more cluster nodes was lost for some period of time during computation.

Here are advices how to avoid mentioned above troubles:

  1. Check and correct settings in DI's Configuration Dialog. This usualy settle 95% of all problems.
  2. Uninstall or disable Antivirus software at all.
  3. Disable or tune Firewall software to allow network activity for DI's executables in bin directory and MPI executables (usually this are mpiexec, smpd in bin directory of MPI installation directory).
  4. Check step by step is MPI installed correctly using sections Setting up DeinoMPI on Windows Cluster, Setting up MPICH2 on Windows Cluster, Setting up LAM/MPI on Linux/MacOS Cluster.
  5. MPI do not support network disconnections between cluster nodes during computation. That is why network connection must be provided continuously.
